# Hermes AI Tier Switcher

Swap the inference backend Hermes uses — driven by GUI choices written to `/etc/serverstick/tier.env`.

## The three tiers

### Local (`tier=local`)
- Uses llama.cpp server, model selected by `llmfit-scan` skill
- Reads `HERMES_MODEL_PATH`, `HERMES_GPU_LAYERS`, `HERMES_CTX_SIZE` from `/etc/serverstick/tier.env`
- Install: `apt install llama.cpp` or use bundled binary
- Start: `systemctl enable --now llama-server`
- NemoClaw env: `INFERENCE_PROVIDER=local`, `INFERENCE_LOCAL_URL=http://localhost:8080`

### BYO (`tier=byo`)
- User pastes their own OpenAI / Anthropic / Groq / OpenRouter key
- Hermes uses OpenAI-compatible API mode
- NemoClaw env: `INFERENCE_PROVIDER=openai`, `OPENAI_API_KEY=*** `OPENAI_BASE_URL` (e.g. `https://api.openai.com/v1` or `https://api.anthropic.com`)

### Managed (`tier=managed`)
- XMR mining on the device funds a TokenRouter account
- Pi Agent runs XMR miner in background, accumulates credits
- Hermes uses TokenRouter API (serverstick's own router)
- NemoClaw env: `INFERENCE_PROVIDER=openai`, `OPENAI_API_KEY=$TOKENROUTER_KEY`, `OPENAI_BASE_URL=https://api.tokenrouter.com/v1`

## What it does
1. Read `tier.env` (written by Svelte GUI)
2. Stop Hermes: `nemohermes serverstick stop`
3. For `local`: start llama.cpp server with chosen model
4. For `byo`/`managed`: write NemoClaw env to `/sandbox/.hermes/.env`
5. Run `nemohermes credentials reset OPENAI_API_KEY` (or local equivalent)
6. Start Hermes: `nemohermes serverstick start`
7. Verify: `curl -H "Authorization: Bearer *** /api/hermes/health` shows healthy

## Files
- `/etc/serverstick/tier.env` — generated by Svelte GUI
- `/etc/serverstick/hermes.token` — Hermes bearer token
- `/sandbox/.hermes/.env` — inside NemoClaw sandbox (Hermes reads this)
